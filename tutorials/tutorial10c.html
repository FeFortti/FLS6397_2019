Mapas no R com pacote sp

Pontos, linhas, polgonos e classes sp

Em nossos exemplos utilizamos data frames para armazenar as infomaes espaciais. Isso foi possvel por que, ao trabalharmos com latitude e longitude de pontos, ficamos limitados a um nico sistema de coordenadas e tampouco precisamos de mais de uma linha do data frame para armazenar o desenho de um objeto (ponto) em um mapa.

H vrios sistemas de coordenadas utilizados em diferentes bases de dados cartogrficas. Assim, por um lado, precisamos de mtodos que nos permitam transformar um sistema em outro para combinar diferentes fontes de dados. Dados especiais armazenados em data frames no contm referncia alguma ao sistema de coordenadas ao qual pertencem.

Por outro lado, se precisarmos de mapas que incluem outras representaes geomtricas, como linhas e pontos, somos obrigados a multiplicar o nmero de linhas em um data frame pelo nmero de vrtices de tais figuras (2 vezes para uma linha e n vezes para um polgono, onde n o nmero de vrtices do polgono). Esse mtodo, ainda que possvel, muito ineficiente.

A soluo trabalharmos com classes de objetos mais complexas (e completas) do que data frames. Vamos falar sobre isso ao explorarmos as principais classes do pacote sp. Em nosso tutorial vamos trabalhar com as classes de objetos do pacote sp e com as funes disponveis para sua manipulao.

Antes disso, por, vamos conhecer o formato mais comum para armazenar dados espaciais shapefile e como abr-los em R.

Para adicioanrmos essa informao ao objeto de dados espaciais, basta fazermos um left_join com os dados em @data:

  mutate(ID = as.numeric(ID), COD_IBGE = as.numeric(COD_IBGE),

Nada de novo, exceto o fato de que o data frame est dentro de um objeto mais complexo. Vamos agora plotar o ano de elaborao dos Planos Diretores de cada municpio como cores nos mapas:

Para adicionarmos novas variveis utilizando o ggplot2, porm, precisamos reorganizar nossos dados. Infelizmente, se simplemente adicionarmos uma varivel que esteja em @data ao argumento fill, por exemplo, obteremos um mensagem de erro.

A soluo ser reorganizar os dados de polgonos em um data frame novo com a funo fortify, perdendo justamente a eficincia que as classes de objetos do pacote sp forneceram.

Na sequncia abaixo, criaremos rmsp_df com a funo fortify. rmsp_df ter apenas variveis espaciais. Para adicionar as demais variveis que esto em @data, teremos de criar um id comum a ambos os data frames e realizar um left_join:

rmsp_df <- fortify(rmsp)
## Regions defined for each Polygons
rmsp_df$id <- as.numeric(rmsp_df$id)
rmsp@data$id <- 0:(nrow(rmsp@data)-1)
rmsp_df <- left_join(rmsp_df, rmsp@data, by = "id")
